Quiz 2: Full Cycle App with Backend

Quiz II Assignment (15%)



Full Stack Mobile Application with Python Backend
Objective


Build a complete end-to-end mobile application system consisting of:

Python foundation skills (verified via certificate)

A FastAPI backend

A MongoDB database

An Android mobile application (Java)



This assignment ensures you can design, implement, and connect all layers of a modern application stack.

Learning Outcomes


By completing this assignment, students will be able to:

Develop Python proficiency for backend development

Design and implement REST APIs using FastAPI

Model and store structured data in MongoDB

Integrate Android apps with backend services

Apply MVVM concepts and asynchronous networking

Build original, working systems (not copy-based)

🧩 PART 1 — Python Foundations (1.5–2 hours)


Task
Complete the Kaggle Python Course:

https://www.kaggle.com/learn/python

Focus on:

Variables and types

Lists and dictionaries

Loops and conditionals

Functions

Download your certificate (with your name)

Deliverable
Upload certificate (PDF or screenshot)

Be ready for TA oral verification

Backend Requirements


1. FastAPI Application


You must implement:

POST /items → Create item

GET /items → Get all items

GET /items/{id} → Get single item

PUT /items/{id} → Update item

DELETE /items/{id} → Delete item

2. Data Model (Must Be Customized)


Each student must define at least 4 fields, including:



Example (Task Manager):

{
  "title": "Submit assignment",
  "description": "Android + FastAPI",
  "priority": "High",
  "image": "img123.png",
  "due_date": "2026-04-10"
}
⚠️ Students MUST:

Add 1 unique field (e.g., difficulty, rating, category)

Explain it in submission

Image field can be skipped, but you won't get the full grade (reference Rubric).

3. MongoDB Integration
Use MongoDB (local or Atlas)

Store all data in a collection

Use _id properly

Convert ObjectId to string

4. Validation


Use Pydantic models:

Required fields

Data types

Default values where applicable

5. Bonus (Required for A grade)


At least ONE of:

Filtering endpoint

Search endpoint

Sorting

Pagination

Image and Camera Integration.

🧩 PART 3 — Android Mobile App (3 hours)
🎯 Requirement


Build a mobile app that connects to your backend.

Mandatory Features


1. List Screen
Display all items from API

Use:

ListView OR RecyclerView

Data must come from backend (not hardcoded)

2. Add Screen
Form to create new item

Must include:

Minimum 3 inputs

Sends POST request to API

3. Detail Screen
Display item details

Allow:

Update OR Delete

4. Architecture Requirements
Use MVVM OR structured separation

Use your preferred navigation

Use Fragments with common ModelView, and one or more fragment will require its own MV (explain why when used)

Use:

Volley OR Retrofit

Use:

Model class (POJO)

5. UI Requirements
Must include:

At least 1 custom layout of your choice

Proper spacing and alignment

Each student must:

Come up with their color theme OR layout variation (no copying from others)
